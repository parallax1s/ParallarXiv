# AGENTS.md

## Scope
These guidelines apply to the entire repository.

## Commit Messages
- Use short, imperative messages (e.g., "Add feature X").
- Reference issues if applicable.

## Testing
- Run `python -m unittest` from the repository root. All tests should pass before committing.

## Formatting
- Keep Markdown files readable. Use two-space indentation for HTML if modifying `index.html`.

## Pull Requests
- Provide a short summary of changes and any manual testing performed.
